Module tomatic

Tomatic is a library that helps to add automatic configuration capabilities on Python programs without bloating files created for store settings with unnecessary lines of code. It has originally designed for use with Django applications acting in settings.py but could be used with any other Python program.

Buckets are simple interfaces for KEY/VALUE repositories.

Buckets

Buckets are simple interfaces for KEY/VALUE repositories that only handle with two specific tasks:

  1. Where KEY/VALUE pairs are stored.
  2. How to retrieve them.

Tomatic comes with two buckets by default:

  • DummyBucket : It's a test purpose bucket and to be use for make sure that your changes didn't break your code. A dummy bucket returns None VALUE for any KEY.
  • EnvironBucket : It's a bucket that uses operating system environment variables to store KEY/VALUE data. Create variables using «profile»__«key»=«value syntax and this bucket will find them.

How to use

Setup Tomatic

Load Tomatic module and the bucket class that you going to use in your code:

from tomatic import Tomatic
from tomatic.buckets import EnvironBucket

Create a Tomatic instance for your settings:

t = Tomatic(EnvironBucket, static_profile="HOMOLOG")

A profile is a name that identifies which set of data you going to use, it's like a directory on file system.

You can turn profile selection more dynamic using env_profile instead of static_profile.

t = Tomatic(EnvironBucket, env_profile="CURRENT_PROFILE")

And set profile value directly on your operation system:

export CURRENT_PROFILE="HOMOLOG"

Setting data

Using EnvironBucket you need to populate environment variables on your operating system as well:

export HOMOLOG__API_HOST='127.16.20.13'
export HOMOLOG__ITEMS_PER_PAGE='50'
export HOMOLOG__DEBUG='true'

Using data

To set them in your program, use:

API = "http://{host}/api/v1".format(t.API_HOST or "localhost")

The or statement use is optional but it works providing default values for cases when a KEY isn't set. In this case if HOMOLOG__API_HOST there isn't defined, "localhost" is the used.

Forcing data types

In Python environment variables are always strings, but you can force a specific data type conversion adding "__«type»__" as a suffix. To force ITEMS_PER_PAGEto be an integer, use:

ITEMS_PER_PAGE = t.ITEMS_PER_PAGE__int__ or 15

Supported types are:

  • Boolean (__bool__)
  • Floating Point (__float__)
  • Integer (__int__)
  • JSON (__json__)
  • String* (__str__).

(*) Default type.

Handling with empty values

In Python a False or True statement results True, and the same happens with 0 or 1, "" or "name" etc… So, to avoid this behavior use fix(tomatic value», «default_value») function…

from tomatic import Tomatic, fix
...
DEBUG = fix(t.DEBUG, True)

To force use of value that came from Tomatic even when empty and to use a default value only if None is returned.
